# 002 - 0.2.0 reliability, correctness, and packaging hardening

- Status: Accepted
- Date: 2026-07-02
- Supersedes: none (extends ADR 001)

## Context

`@naughtyjs/proberesponder` reached feature parity with the Go original in
0.1.x. A world-class review of the package surfaced correctness, reliability,
performance, and distribution gaps that are important for a library used on the
critical path of Kubernetes liveness/readiness probing.

## Decision

Ship a 0.2.0 release that hardens the package without breaking the core
`ProbeResponder` API. The package remains ESM-only.

### Content negotiation (RFC 9110)

The HTTP extension previously treated an absent `q` parameter as quality `0`,
matched media types by substring, and ignored wildcards. This produced
order-dependent, non-compliant selection.

New behavior:

- Absent `q` defaults to `1.0`.
- `*/*` and `type/*` wildcards are supported.
- Media types are matched exactly (type + subtype).
- Ties are broken by an explicit server preference: JSON > HTML > plain > XML.
- If nothing is acceptable (all `q=0`), the response falls back to JSON.

This is a behavior change and is why the release is a minor (0.x) bump.

### HTTP server reliability

- Default `headersTimeout`/`requestTimeout` raised from `1000ms` to `10s`/`15s`.
  A sub-second request timeout caused spurious probe failures under GC pauses
  and node load. Timeouts are now configurable via `ServerOptions`.
- Routing uses an O(1) route map instead of a per-request linear scan.
- Route handlers may be async and are wrapped in an error boundary that
  responds `500` (when headers are unsent) instead of leaving a hung socket.
- Responses set `Cache-Control: no-store`, `Content-Length`, and
  `charset=utf-8`.

### Dependency prober reliability

- `start()` options detection is now structural (`isProber` type guard), so a
  probe can never be misclassified as a trailing `StartOptions` object,
  regardless of incidental fields. This removes the previous `"unref" in obj`
  duck-typing footgun.
- `withTimeout` attaches a rejection handler to the check promise so a checker
  that ignores its `AbortSignal` and rejects after the timeout cannot surface
  as a process-crashing `unhandledRejection`.
- A non-positive timeout means "no timeout" (run directly).
- `probeDependencies` stamps a whole batch with a single `asOf` instant.
- `stop()` is idempotent and prevents further scheduling.

### Packaging / distribution

- `exports` entries list `types` first (required for `node16`/`nodenext`), plus
  top-level `types`/`module` fallbacks and a `./package.json` export.
- Sourcemaps are no longer published (smaller install).
- `publint --strict` and `@arethetypeswrong/cli` gate CI, the release workflow,
  and `prepublishOnly`. An `.attw.json` records the deliberate ESM-only profile.
- Added `keywords`, `funding`, and `CHANGELOG.md` to the published files.

## Alternatives considered

1. Dual ESM + CJS build.
    - Pros: broadest consumer reach.
    - Cons: contradicts ADR 001's ESM-only intent, larger surface to validate.
    - Decision: stay ESM-only; document dynamic `import()` for CJS consumers.

2. Keep the sub-second HTTP timeouts.
    - Pros: none meaningful for probe endpoints.
    - Cons: self-inflicted probe flakiness under load.
    - Decision: raise defaults, make them configurable.

3. Keep permissive/loose content negotiation for backwards compatibility.
    - Pros: no behavior change.
    - Cons: non-compliant and order-dependent; wrong for clients that rely on
      `q` semantics.
    - Decision: adopt RFC 9110 semantics; call it out in the changelog.

## Consequences

Positive:

- Correct, standards-compliant negotiation.
- No process crashes from misbehaving dependency checks.
- Robust probe endpoints under real-world load.
- Verifiable, minimal, correctly-typed published package.

Negative:

- Content-negotiation selection changes for `Accept` headers that omit `q`.
- One defensive `switch` default branch and a couple of timing-dependent
  branches remain uncovered by design.
